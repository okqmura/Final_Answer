- 次の仮想マシンとdockerとの違いについての文のうち、誤っているものを答えなさい
  1. dockerは、ホストOSのカーネルをそれぞれのコンテナで利用するため、起動が速い
  2. dockerは、複数のコンテナで構成されるため、root権限を奪取されても影響が出にくい
  3. dockerは、プロセスごとにコンテナを立ち上げるため、環境に依存せず動作する
  4. dockerは、ファイルで環境の情報を共有できるため、他者と同一の環境を構築しやすい
  →2 
  DockerコンテナはホストOSのカーネルを共有するため、一つのコンテナがroot権限を奪取すると他のコンテナやホストOSに影響を及ぼす可能性があります
  https://qiita.com/k_h_mirai/items/cd62955c4a4ec092e8f6

- 次のdocker composeについての文のうち、正しいものを選びなさい
  1. docker composeは複数のコンテナを一つのコンテナにまとめることができる
  2. docker composeはアプリケーションに必要なコンテナをまとめて立ち上げることができる
  3. docker composeで複数のコンテナを作成する際には、docker fileも一つでいい
  →2
   Docker Composeとは、複数のコンテナを効率的に操作するためのツールです。Docker Composeを使う場合、「compose.yaml」という名前のファイルに、各コンテナに対する設定をあらかじめ定義しておきます。その上で専用コマンドを実行することにより、定義した内容に従い複数コンテナを一括で同時に起動させることができるのです。
